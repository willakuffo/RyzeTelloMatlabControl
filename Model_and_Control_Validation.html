Model and Control Validation with Estimated Altitude Model - Integration with Simulink

Model and Control Validation is done in the (Tello Altitude Model Validation and Control) TAMVC.slx. This simulink model implements 2 control loops in parallel for real time simulation and control of both the physical plant (the actual ryze tello drone) and the estimated dynamic altitude model for model and control validation.
  • The first control loop is an HIL involving the physical Tello drone plant
  • The second control loop invloves the estimated dynamic model of the tello drone as transfer function

The simulink model implements a simple altitude reference tracking closed loop control using a P - controller. For each control loop, there are the following blocks;
  • Reference altitude : constant block
Desired altitude to be tracked
  • If statement block and if action subsytem block
TYpically without this, the computation for the simulation of the dynamic model is faster than the plant. This may happen because the real plant needs some time to initialize and prepare itself for take off. These blocks ensure that the two loops are aligned in time. It does this by allowing the 2nd control loop to run only when the physical plant is just about to take off consequently ensuring that the simulation of the dynamic altitude model runs in real time along side the HIL.

Converts the reference altitude from centimeters to meters

  • Saturation block
Limit the control effort to within rated bounds from -100 to 100
  • Delay block :
Required to break the HIL algebraic loop of the HIL as it contains the physcial tello plant (which is a dicrete time model sampled at T = 0.1s) with a direct feedthrough. The unit step delay eliminates potential errors associated with this when the simulink solver fails to converge at a particular timestep. Setting up and Understanding the Simulink Integration

It is extremely simple to create a simulink model for simulation purposes; however, in order to achieve HIL with the Ryze Tello Drone as well as simulation of its dynamic altitude model, the TelloControl Library developed in matlab is used in the simulink integration with the Tello. The following steps were taken to achieve this integration, mostly because MATLAB's code generation does not support several of the implementations used in the TelloContol Library.
Model Properties
Under model properties, 2 callback functions are utilized. To learn more about these callbacks see here.
  • initFcn
In this callback, the Tello is initialized and prepared for takeoff. The dynmanic altitude model is also loaded into the base workspace. See here to learn more about Matlab workspaces.
  • StopFcn
In this call back the controller is made to land the drone after the simulation time ends.
The Physical Plant
The physical Tello Plant (Drone) in the HIL is implemented as a simulink user defined matlab function. This is found in the HIL (first) control loop. See the figure below for its implentation.
Discrete TIme Dynamic Altitude Model
After the initFcn callback has loaded the tello_altitude_discrete.mat file which is the discrete time altitude model into the workspace, it returns two variables ad1 and bd1 in this case. The former are the coefficients of the estimated transfer function numerator and the latter being the coefficients of the estimated transfer function denominator. Notice the sample time being 0.1 which describes the 10Hz sampling frequency of the onboard SDK.
