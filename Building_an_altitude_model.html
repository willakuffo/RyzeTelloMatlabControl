Building a Dynamic Altitude Model with Data

This version of the software currently implements control for only the altitude of the drone. The trackpaad or mouse is used as the controller for the drone's altitude. The control input is generated from the y-coordinate (up-down) of the cursor position on the live plot.
A dynamical altitude model can be built from the control input and the recorded output of the drone.

Typical Workflow

Collecting data
To develop a dynamc altitude model for the Ryze Tello Drone, run this script. This is the tello_altitude_rc.m
This script collects and visualizes in real time the reported altitude of the drone for 100 samples after which it is saved as a .mat file.
clear;clc;close all;
tc = TelloControl;
 
tc.initTello();
tc.getTelloState()
 
 
tso = TelloStateObserver(tc,"observers",{'h'},'collectors',{'h','vgz','agz'},limitAltitude=200,n_samples=100);
%tc.motoron();
 
[collected,observed,input] = tso.stateObserve();
 
disp('saving state collected state variables and input...')
telloStateData = {};
telloStateData.states = collected;
telloStateData.input = input;
 
save('tellosStateData.mat','telloStateData')
 
System Identification
Using matlab's system identification app, a dynamical altitude model of the drone can be estimated as a transfer function or a state space model. Other model types can also be explored.
Before this is done, lets take a look at the nature of the data we saved in the .mat file
Loading the .mat file
This returns a structure with field telloStateData
data = load('tellosStateData.mat')
data = struct with fields:
telloStateData: [1×1 struct]
Within the telloStateData field contains the fields
'states' : structure containing collected state variables. In this case it will contain the specified collectors {'h','vgz','agz'} as the recrded states. Each being a field
'input' : the control input to the drone saturated between -100 and 100

hold off
Run the systemIdentification command to estimate an altitude model for the drone using this data
systemIdentification
For more information on how to use matlab's system identification app see, this video
It is recommended to estimate a distcrete time model with a sampling time of 0.1s. According to SDK documentation, the onboard sdk of the ryze tello dronereleases the state variables at a frequency of 10Hz
Also use a sampling time of 0.1s when importing the data into the system identification app.
